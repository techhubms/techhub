aycabas announces the IQ Series: Foundry IQ, a set of developer-focused episodes (with videos, Jupyter notebooks, and a GitHub repo) that walks through building knowledge-centric AI systems where agents query structured knowledge bases across sources like Azure AI Search, Blob Storage, Fabric/OneLake, SharePoint, and the web.<!--excerpt_end-->

## Overview

The **IQ Series: Foundry IQ** (starting **March 18, 2026**) is a developer-focused set of episodes and companion materials (expert videos, **Jupyter notebooks**, doodle recaps, and a **GitHub repository with cookbooks**) aimed at helping developers build **knowledge-centric AI systems** using **Foundry IQ**.

Explore the series and materials:

- https://aka.ms/iq-series

## Why this series (and why “knowledge” matters for agents)

The post frames **AI agents** as a growing way to build applications, but notes that agents only become useful when they can reliably access **knowledge and context** about the world they operate in.

The series focuses on how modern AI systems:

- Retrieve information
- Reason across sources
- Synthesize answers
- Orchestrate multi-step interactions

A key theme is that **retrieval isn’t treated as a single one-off pipeline step**. Instead, **Foundry IQ** treats knowledge as something the system works with throughout the reasoning process.

## What is Foundry IQ?

**Foundry IQ** is presented as a way for AI systems to work with knowledge in a more structured, reusable way.

Instead of wiring retrieval logic into every application:

- Developers define **knowledge bases**
- Knowledge bases connect to **documents, data sources, and other information systems**
- AI agents query these knowledge bases to gather context needed to:
  - Generate responses
  - Make decisions
  - Complete tasks

The intended benefit is that knowledge can be **organized, reused, and combined across applications**, rather than rebuilt per scenario.

## What’s covered in the IQ Series?

The series is released as **three weekly episodes**:

- **Foundry IQ: Unlocking Knowledge for Your Agents — March 18, 2026**
  - Introduces Foundry IQ and the core ideas behind it.
  - Explains how agents work with knowledge.
  - Walks through the main Foundry IQ components that support knowledge-driven apps.

- **Foundry IQ: Building the Data Pipeline with Knowledge Sources — March 25, 2026**
  - Focuses on **Knowledge Sources** and how content flows into Foundry IQ.
  - Mentions sources including **SharePoint**, **Fabric**, **OneLake**, **Azure Blob Storage**, **Azure AI Search**, and the **web**.

- **Foundry IQ: Querying the Multi-Source AI Knowledge Bases — April 1, 2026**
  - Focuses on **Knowledge Bases**.
  - Shows organizing multiple knowledge sources behind a single endpoint.
  - Demonstrates querying across sources and synthesizing results for complex questions.

Each episode includes:

- A short executive introduction
- A tech talk going deeper on the topic
- A visual recap with doodle summaries
