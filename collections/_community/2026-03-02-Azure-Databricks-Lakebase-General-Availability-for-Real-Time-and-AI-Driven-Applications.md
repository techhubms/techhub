# Azure Databricks Lakebase: Now Generally Available

**Author:** AnaviNahar

Azure Databricks Lakebase has officially reached general availability, enabling teams to build and validate real-time and AI-driven applications directly on their lakehouse foundation using Azure Databricks.

## Why Azure Databricks Lakebase?

Lakebase addresses the needs of modern applications that demand intelligent operation, real-time data access, and robust governance. Its serverless, Postgres-compatible architecture separates compute from storage, allowing on-demand scalability and cost efficiency. Key features include:

- **Serverless Postgres for Production**: Managed Postgres environment with built-in reliability and production-ready performance, abstracting infrastructure management.
- **Instant Branching and Point-in-Time Recovery**: Create zero-copy branches for test/dev or analytics; easily restore databases to precise snapshots for error recovery.
- **Unified Governance with Unity Catalog**: Apply the same fine-grained data governance policies across operational, analytic, and AI workloads, ensuring compliance and auditing.
- **AI-Ready and Real-Time Support**: Designed to serve AI patterns like real-time feature serving, agent memory, and low-latency app state, seamlessly integrating with the analytics lakehouse.

The ability to operate on governed, lake-backed data directly simplifies architecture by reducing ETL and storage duplication.

## Core Use Cases

- Building real-time applications using lakehouse data
- Powering AI agents and features with persistent, governed memory
- Accelerating release cycles with safe, isolated database branches
- Simplifying the modern data stack with fewer moving parts

Developers interact with Lakebase through familiar Postgres interfaces and tools, reducing adoption friction.

## Getting Started

Azure Databricks Lakebase is natively integrated with Azure Databricks workspaces. To provision, users simply add Lakebase to their workspace, making it accessible for new intelligent app development. For further details and to start building, see the [official documentation](https://learn.microsoft.com/en-us/azure/databricks/oltp/projects/).
