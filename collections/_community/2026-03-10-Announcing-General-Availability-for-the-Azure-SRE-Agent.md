# Announcing General Availability for the Azure SRE Agent

Azure SRE Agent has reached General Availability (GA), bringing a powerful AI-driven operations teammate to organizations looking to improve uptime, accelerate incident response, and reduce operational toil on Microsoft Azure. Informed by extensive customer feedback, this platform unlocks new ways to automate and standardize cloud operations at scale.

## What’s New in the GA Release

- **Deep Context Awareness:** Azure SRE Agent now operates with team-specific context, including loaded source code, operational tooling, and architectural knowledge. This design provides faster, more accurate diagnostics tailored to real environments.
- **Built-In Computation:** With a native code interpreter, the agent can execute Python to generate reports, dashboards, and exports (PDF, Excel, charts) directly from operational data.
- **Evolving Memory:** The agent builds a knowledge base with every interaction. Teams can explicitly teach the agent, preserving tribal knowledge for future automation and troubleshooting.
- **Expanded Automation:** GA introduces new automation modules: skills, subagents, Python tool support, and agent hooks, alongside richer ecosystem connectivity via MCP connectors.

For detailed release notes, see the [what's new documentation](https://aka.ms/sreagent/new).

## How Azure SRE Agent Benefits Operations

Azure SRE Agent acts as an AI-powered teammate for:

1. **Accelerated Root-Cause Analysis**: The agent automates multi-layer investigation across apps, platforms, and infrastructure, correlating telemetry and clearly explaining root causes. This shortens Mean Time to Resolution (MTTR) and reduces on-call burden.
2. **Incident Response with Knowledge Reuse**: It leverages knowledge from prior incidents and runbooks, reducing repetitive work and supporting confident, consistent responses over time.
3. **Automated Mitigation, Governed**: Teams choose autonomy levels, from actionable recommendations to full automation within guardrails, helping maintain compliance and operational control.
4. **Standardized Operations at Scale**: Integrates with existing tools, policies, and runbooks, scaling automation consistently across teams and reducing integration overhead.
5. **Proactive Reliability**: Identifies misconfigurations and emerging issues early so teams can address risks before they impact reliability.

**Customer highlight:** [Ecolab](https://aka.ms/sreagent/ecolab) reduced daily performance alerts from 30-40 to under 10 and sped up root-cause analysis with Azure SRE Agent, freeing teams to focus on strategic reliability improvements.
