## Azure Local Disconnected Operations: Running Sovereign Cloud, Productivity, and AI in Air‑Gapped Environments

Many organizations operate in environments where **continuous cloud connectivity isn’t possible—or isn’t allowed**. Sovereignty requirements, regulatory constraints, security policies, and physical isolation can require infrastructure to run **fully disconnected (air‑gapped)** while still providing modern “cloud-like” capabilities.

In this post and accompanying video, Azure Local **Disconnected Operations** are introduced as a way to run and manage Azure Local **without dependency on continuous connectivity to Azure public cloud**.

## What are Azure Local Disconnected Operations?

**Azure Local Disconnected Operations** enable organizations to deploy and manage **Azure Local** in:

- Fully disconnected environments
- Air‑gapped environments

The key idea is that **data, control, and operations remain local**, allowing operations inside an organization’s security boundary while keeping a consistent Azure-style operational approach.

Azure Local is described as providing a **local control plane** for operating infrastructure on customer-owned hardware.

## Where this matters

The post calls out scenarios where disconnected operations are especially relevant:

- Government and defense
- Critical national infrastructure
- Regulated industries (for example: healthcare and finance)
- Remote or isolated locations with limited connectivity
- Business continuity and disaster recovery

## Beyond infrastructure: “Sovereign Private Cloud” stack

Disconnected operations are positioned as part of a broader **Sovereign Private Cloud** approach spanning multiple layers:

### Azure Local (infrastructure)

Run locally, without cloud dependency:

- Virtual machines
- Kubernetes (AKS / Azure Kubernetes Service)
- Selected Azure Arc-enabled services

### Microsoft 365 Local (productivity)

Run **core productivity services** locally in disconnected environments to support collaboration and business continuity without relying on the public cloud.

### Foundry Local (AI workloads)

Run **AI models and inferencing workloads locally** for sovereign AI scenarios where data must not leave the environment.
