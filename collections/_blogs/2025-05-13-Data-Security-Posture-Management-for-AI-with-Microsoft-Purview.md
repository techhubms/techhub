# Data Security Posture Management for AI

**Author:** Petrus Vasenius

Microsoft Purview Data Security Posture Management (DSPM) for AI is now generally available, providing organizations with advanced tools to monitor, secure, and govern data used by AI applications. This post explains how DSPM integrates with various Microsoft and non-Microsoft AI solutions to strengthen data compliance, security, and risk mitigation.

## Key Features and Benefits

- **Centralized Dashboard:**
  - The overview dashboard provides visibility into AI usage, sensitive data interactions, and activities across Microsoft Copilot, third-party AI apps (including Google Gemini, ChatGPT, Copilot for Bing), and more.
  - Security analytics and recommendations are presented clearly for ongoing monitoring and quick insights.

- **AI Data Security Recommendations:**
  - The recommendations blade tracks actionable guidance, statuses (not started, dismissed, completed), and includes preview features.
  - Each recommendation offers remediation descriptions and policy suggestions directly from Microsoft Purview.

- **Comprehensive Reports:**
  - The reports section summarizes results of default and custom-created security policies.
  - Reports are categorized by AI application type, such as Microsoft Copilot Experiences and Enterprise AI apps.

- **Policy Management:**
  - Manage and monitor one-click and custom AI-related policies directly in the Purview portal.
  - For advanced policy actions, integration with solutions like Communication Compliance allows remediating unethical behavior in Copilot.

- **Activity Explorer:**
  - Detailed activity logs include types, users, dates, application categories, and detection of sensitive information.
  - Visual timelines display sensitive data and AI interactions for compliance auditing.

- **Data Risk Assessments (Preview):**
  - Identifies oversharing and data access risks, offering fixes to enhance data protection.
  - Automatically assesses top SharePoint sites weekly, regardless of custom assessments.

## Implementation Approach

To maximize data security posture when adopting AI, the article recommends a workshop-driven onboarding for Purview DSPM, mapping responsibilities, data identification, and governance actions. The workshop guides organizations step-by-step through enabling and configuring DSPM tailored to their unique requirements.

## Conclusion

Microsoft Purview DSPM for AI enables organizations to safely leverage AI innovations while adhering to best practices for data security and regulatory compliance. By using dashboards, actionable recommendations, and integrated policy management, businesses can oversee AI interactions with sensitive data and ensure accountability, all orchestrated from within the Microsoft security ecosystem.
