# Azure Update - Friday the 13th February 2026

**Presented by John Savill's Technical Training**

Stay up to date with the latest Azure news, features, and enhancements. This update covers:

## Table of Contents

- Azure Monitor pipeline improvements
- Vaulted backups for Azure Disk
- AKS (Azure Kubernetes Service) App Gateway for Containers add-on
- AKS support for Kubernetes 1.34
- Azure Front Door Premium with Private Link origin in UAE North
- Claude Opus 4.6 model availability on Azure Databricks
- Databricks Supervisor Agent introduction
- Azure SQL Database support for 1-4 named secondaries
- Azure expands to the new Thailand South region
- Kimi-K2.5 available in Azure AI Foundry
- Entra (Azure AD) SoA (Separation of Administrator) user management change

## Highlights

### Azure Monitor Pipelines

Discover updates to Azure Monitor pipelines, enabling more robust monitoring solutions and simplified operational workflows for Azure environments.

### Azure Disk Vaulted Backups

Explore the new vaulted backup options for Azure Disks, improving data protection and retention strategies within your cloud workloads.

### AKS: App Gateway for Containers Add-On and K8S 1.34 Support

Unpack the new App Gateway for Containers add-on in AKS. Now also supporting Kubernetes 1.34, AKS keeps up with the latest container orchestration standards and ecosystem tools.

### Networking and Regions

- AFD Premium introduces Private Link origins in the UAE North region for secure web application delivery.
- Microsoft Azure establishes a presence in the new Thailand South region, further expanding global cloud infrastructure.

### Data Platform Updates

- New features in Azure Databricks, including the Claude Opus 4.6 model and Supervisor Agent.
- Azure SQL Database now supports between 1 and 4 named secondary replicas for enhanced data resilience and high availability.

### AI Services and Security

- The Kimi-K2.5 model is now integrated into Azure AI Foundry, providing more options for AI workloads.
- Entra (formerly Azure AD) SoA user management change introduces improvements in identity and access management.
