# Manage All GitHub Copilot Agents with Mission Control

**Presented by:** Andrea Griffiths and Ellie Bennett (GitHub Senior Product Manager)

## Overview

GitHub introduces Mission Control, a new feature of GitHub Copilot that allows developers to centrally manage, monitor, and steer all their Copilot-powered coding agents within a unified interface. Mission Control is designed for increased productivity and enhanced control across your development workflows.

## Key Features Covered

- **What is Mission Control?**
  - Provides a central hub for tracking and controlling all GitHub Copilot coding agents.
  - Visualizes ongoing tasks, agent actions, and allows mid-task interventions.

- **Starting and Managing Tasks**
  - Use the agents panel to initialize and track tasks.
  - View detailed session logs and code diffs for better transparency.

- **Real-Time Steering**
  - Demo of how to adjust or correct Copilot agent actions in real time.
  - Enables interactive engagement to guide agents if their output veers from the original intent.

- **VS Code Integration**
  - Seamlessly switch from Mission Control on the web to the VS Code editor to continue work without disruption.

- **Custom Agents and Workflow Flexibility**
  - Support for custom agent definitions tailored to specific projects or needs.
  - Demonstrates smooth handoff between mobile and web platforms.

- **Productivity Gains and Roadmap**
  - Discussion on productivity improvements realized with Mission Control.
  - Insights into upcoming features and future directions.

Instructions are given on early access and trying the new Mission Control capabilities.
