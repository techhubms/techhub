# Powering Azure Fabric and AI with Eon’s Data Lake

## Overview

This Microsoft Ignite session explores Eon’s cloud-native data lake solution on Azure, highlighting how backup data becomes instantly usable as Iceberg tables in Blob Storage. The presentation demonstrates how this enables direct analytics and AI innovation in Microsoft Fabric and OneLake environments.

## Key Topics Covered

- **Eon as a Cloud Backup Posture Management Platform**: Strategies for managing backups as active, queryable data.
- **Live Data Access & Cost Optimization**: Techniques to make backup data directly queryable, improving usability and reducing infrastructure costs.
- **Integration with Fabric Engines**: Backup data is unified and made accessible to Microsoft Fabric engines including SQL, Spark, and KQL for analytics workloads.
- **AI Innovation on Azure**: Explore ways backup data fuels AI activities using Azure OpenAI and Fabric features.
- **Demo Highlights**:
  - Navigating Eon’s Inventory Page across cloud providers
  - Content-based data classification for sensitive information
  - Monitoring protected resources in the Eon dashboard
  - Performing full/instance-level or disk-level restores across regions
  - File indexing and search capabilities out of the box

## Collaboration and Security

Learn how Eon improves collaboration by unifying data protection, analytics, and AI operations on a single data lake architecture. Discover security measures, data classification, and live monitoring practices built into the platform.

## Practical Use Cases

- Immediate analytics on backup data for insights and reporting
- AI-powered workflows for data discovery using Azure OpenAI
- Cross-cloud restoration and compliance with sensitive information monitoring
- Efficient indexing and search across disparate backup sources
