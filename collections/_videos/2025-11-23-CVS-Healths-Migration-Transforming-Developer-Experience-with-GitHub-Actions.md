# CVS Health’s Migration: Transforming Developer Experience with GitHub Actions

**Speakers:** Irene Modeste (Sr Product Manager - Developer Experience), Jim Beyers (VP, Developer Experience), Mitch Enderle (Executive Director, Platform Engineering), CVS Health

## Overview

CVS Health undertook a massive migration moving 150,000 repositories from legacy CI/CD systems to a unified workflow built around GitHub and GitHub Actions. This shift eliminated complex, outdated infrastructure and centralized developer processes for thousands of engineers.

## Key Topics Covered

- **Mission & Technology Strategy:** Introduction to the motivation behind standardizing developer platforms.
- **Developer Experience Improvements:** How a common workflow enhanced consistency, collaboration, and productivity.
- **Platform Unification:** CVS Health’s rationale for moving to GitHub EMU and consolidating tools.
- **Migration Planning:** Approaches to scope, roadmap, and overcome technical and organizational challenges.
- **Lessons Learned:** Insights on practical strategies, what worked well, hurdles encountered, and future directions for CI/CD evolution.

## Migration Highlights

- Legacy tools replaced by scalable GitHub Actions workflows.
- Platform teams transitioned from maintenance to innovation.
- Accelerated collaboration across healthcare development teams.
- Migration roadmap detailed, emphasizing planning and phased execution.
- Real-world results: Improved speed, reliability, and onboarding for developers.

## Practical Takeaways

- Standardizing on modern CI/CD tooling simplifies platform maintenance.
- A unified developer experience can drive both technical and cultural change.
- Effective migration requires upfront planning, adaptable solutions, and continuous support.
