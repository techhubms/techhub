# AI Builder’s Guide to Agent Development with Foundry Agent Service

**Presenters:** Salman Quazi, Dan Taylor, Travis Wilson  
**Event:** Microsoft Ignite 2025 – Advanced (300) Session BRK201  
**Language:** English (US)

## Introduction

This session explores the next era of AI development, focusing on how developers can leverage the Foundry Agent Service to build, operate, and scale production-ready AI agents on Azure.

## Key Topics

- **Agent Authoring:** Step-by-step demo for creating custom agents connected to GitHub and using managed OAuth for authentication.
- **Connecting Tools and Data:** Techniques for integrating enterprise data and third-party tools while ensuring interoperability.
- **Performance Evaluation:** Guidance on running agent evaluations locally and reviewing detailed results via Azure portal.
- **Agent Memory & Personalization:** Features for enabling context-aware interactions, persistent memory, and personalized user experiences.
- **Integration of OpenAI API–Based Projects:** Learn how to migrate existing solutions into Foundry Agent Service with minimal changes, benefitting from Microsoft’s enterprise-grade governance and observability.
- **Model Context Protocol:** Insight into secure data exchange, agent-to-agent capabilities, and multi-agent workflows for complex tasks.
- **LangGraph Tracing:** Demonstration of integrated tracing to debug and optimize agent workflows.
- **Multi-Agent System Design:** Practical tips for building and orchestrating real-world multi-agent bots via workflows and prompt agents, including ticket creation and automated troubleshooting scenarios.

## Demo Highlights

- **GitHub-Connected Agent:** Walkthrough of setup, authentication, and operational flows.
- **Evaluation & Portal Integration:** Local testing and advanced results visualization.
- **Personalization:** How agent memory boosts adaptive responses.
- **Workflow Interaction:** Automated processes showcasing workflow robustness.
