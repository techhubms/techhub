# Modernizing Oracle Workloads with Oracle Database@Azure and Native Azure AI Services

## Session Overview

This session from Microsoft Ignite 2025 details how organizations are modernizing mission-critical Oracle workloads by deploying Oracle Database@Azure directly within Microsoft Azure. Key focus areas include high performance, availability, and security, all backed by Oracle Exadata Database Service and the emerging Autonomous AI Database.

## Key Topics Covered

- **Oracle Strengths on Azure**: Performance, high availability, security, and reliability of Oracle solutions in the Azure cloud environment.
- **Real-Life Architectures**: Features a high-availability, secure architecture built with Oracle Database@Azure, highlighting architectural diagrams and deployment patterns.
- **Data Backup and Storage**: Solutions for data backup and long-term retention, including leveraging Oracle Cloud Infrastructure (OCI) and Azure regions.
- **Customer Case Study – Conduent & EZPass**: Examines how a large-scale tolling system utilizes Oracle’s strengths on Azure for high-security, mission-critical requirements.
- **AI-Driven Insights and New Opportunities**: How combining Oracle’s databases with native Azure AI services empowers organizations to extract value and make data-driven decisions.
- **Looking Ahead: Oracle AI Database 26ai**: Announcements regarding Oracle’s roadmap for data management and next-generation, AI-enhanced capabilities.
- **Autonomous AI Lakehouse**: Introduction of the new Autonomous AI Lakehouse, designed to provide instant, interactive, and secure data conversations by unifying analytical and operational data.

## Practical Learnings

- Steps to migrate and deploy Oracle workloads on Azure while ensuring operational excellence
- Leveraging Azure-native analytics and AI services on enterprise data sets
- Building secure, compliant cloud architectures for highly regulated industries
- Accelerating business transformation by unlocking AI-driven operational and analytical insights
