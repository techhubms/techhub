# PostgreSQL Like a Pro: Azure HorizonDB overview

Dingding Liu shares a sneak peek at **Azure HorizonDB**, described as Microsoft’s newest **managed PostgreSQL** service built for **extreme scale and performance**.

## What Azure HorizonDB is positioned to do

- Target **mission-critical** PostgreSQL workloads at large scale.
- Emphasize performance and resilience features such as:
  - **Decoupled compute and storage**
  - **Sub-millisecond multi-zone commits**
  - Scaling patterns involving **replicas over shared storage**

## AI-oriented capabilities mentioned

Azure HorizonDB is also presented as supporting **AI-native workloads**, including:

- **Built-in vector indexing** (for vector search / AI retrieval scenarios)
- **SQL-based AI functions**
- Integration with **Microsoft Foundry** (likely referring to Azure AI Foundry)

## Demo and workflow highlights (as described)

- Creating/adding a new **HorizonDB cluster in VS Code**
- **Query conversion** (shown as part of the demo flow)
- Discussion of **performance at scale**
- Extending the system by **scaling out replicas over shared storage**
