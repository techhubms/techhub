# Introducing Microsoft Foundry Tools for Agentic AI Apps | BRK192

## Session Overview

This session from Microsoft Ignite 2025 brings forward a new suite of Microsoft Foundry tools focused on empowering agentic AI workflows. These tools are designed to enhance AI agents with:

- Document parsing and content understanding
- Call recording transcription
- Custom translation capabilities
- Bring-your-own-model support (including GPT-4.0, GPT-4.1)
- Easy integration as APIs or MCP (Microsoft Copilot Platform) servers

### Key Topics Covered

- **Live Demo**: Insurance claim processing agent
  - Retrieval of historical data from Azure Databases
  - Claim approval time calculation
  - Use of Cosmos DB for storage and data integration
- **Document-Centric Workflows**
  - Content understanding tools for automating document review and data extraction
- **Model Version Control**
  - Introducing support for different model versions allowing users to select GPT models aligning with their needs
- **Privacy and Security**
  - Redaction of sensitive information in translated documents
  - Data privacy focus throughout the workflow

### Technical Highlights

- **Agentic Solutions**: Designed for seamless integration into AI-driven applications using either API calls or as MCP servers.
- **Production-Ready Tools**: Emphasis on stability, scalability, and plug-and-play usage for enterprise needs.
- **Data Integration**: Demonstration of live agent interacting with Azure-backed databases and Cosmos DB for end-to-end process automation.

### Demo Scenario

The live demonstration showcased an AI agent handling insurance claims, utilizing Foundry tools to:

- Parse submitted documents
- Extract and analyze relevant data
- Access and update database information
- Respond to queries with high-confidence automated reasoning
