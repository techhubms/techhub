# The Security tab is now Security & quality

The top-level **Security** tab across repositories, organizations, and enterprises has been renamed to **Security & quality** on github.com. The goal is to colocate code quality findings alongside security alerts so teams can triage code-related issues from one place.

## What’s changed

- The **Security** tab is now **Security & quality** at the repository, organization, and enterprise levels.
- The repository sidebar section previously labeled **Vulnerability alerts** is now **Findings**.
- The repository sidebar includes a new **Code quality** section showing enablement status.
- The repository **Policy** sidebar item is now **Security policy**.

## What hasn’t changed

- **URLs and API endpoints remain the same**, so you don’t need to update:
  - bookmarks
  - scripts
  - integrations
- The underlying alert types and data are unchanged.

## Availability notes

- This navigation update is available now on **github.com**.
- It is **not included in GitHub Enterprise Server**.
- This change “lays the groundwork” for the upcoming **GitHub Code Quality** general availability launch.
