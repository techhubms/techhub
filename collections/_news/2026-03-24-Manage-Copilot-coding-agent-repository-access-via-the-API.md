Allison announces a public preview of new GitHub REST APIs that let organization owners manage GitHub Copilot coding agent access across repositories, enabling programmatic control over whether the agent is enabled for none, all, or selected repos.<!--excerpt_end-->

## Overview

Organization owners can now manage **GitHub Copilot coding agent** repository access programmatically using the new **Copilot coding agent management REST APIs** (public preview).

- About Copilot coding agent: [Copilot coding agent](https://docs.github.com/copilot/concepts/agents/coding-agent/about-coding-agent)
- REST API reference: [Copilot coding agent management REST APIs](https://docs.github.com/rest/copilot/copilot-coding-agent-management?apiVersion=2026-03-10)

## What you can do with the new REST APIs

With these APIs, you can:

- Choose whether Copilot coding agent is available for:
  - No repositories
  - All repositories
  - Only selected repositories owned by your organization
- When set to **Only selected repositories**, you can:
  - Add allowed repositories
  - Remove allowed repositories

## UI alternative

You can also manage these settings in the GitHub UI:

1. Go to your organization **Settings**
2. Select **Copilot**
3. Select **Coding agent**
