# GitHub Projects Hierarchy View and Issue Form Enhancements

**Author:** Allison

GitHub has released several important updates aimed at making project and issue management more efficient for development teams:

## Hierarchy View Improvements

- **Filter Sub-Issues:** Users can now filter sub-issues with queries such as `sub-issues.is:open` to quickly focus on relevant items.
- **Deduplication of Issues:** Items appearing in multiple places within the project hierarchy are now deduplicated, reducing clutter.
- **Persistence of Expand/Collapse State:** When users adjust the expand/collapse state of sub-issues, their preferences are saved for a consistent view on return.
- **Community Feedback:** Developers are encouraged to join discussions via the GitHub Community for further feedback and questions.

## Issue Forms Now Support File Upload Fields

- When creating new issue forms, maintainers can add upload fields, making it possible (or required) to collect key files like logs, screenshots, and crash reports up front.
- This ensures submitted issues come with all necessary context for quicker troubleshooting.

## Bulk Move Items Between Iterations

- In project views grouped by iteration, it's now possible to bulk move all issues from one iteration to another using the group header menu.
- This feature accelerates rolling over unfinished work or rescheduling items across sprints.

For full details, see the [official changelog post](https://github.blog/changelog/2026-01-15-hierarchy-view-now-available-in-github-projects/) and [GitHub Community discussion](https://github.com/orgs/community/discussions/184225).

These enhancements continue to build on GitHub's focus on project management and developer productivity.
