Allison announces that GitHub Copilot’s third-party coding agents (Claude and Codex) now support model selection on github.com, including the currently available Claude Sonnet/Opus and GPT-5 Codex model options, plus notes on required admin policies and repo settings.<!--excerpt_end-->

# Model selection for Claude and Codex agents on github.com

Model selection is now available for the [Claude and Codex third-party coding agents](https://docs.github.com/enterprise-cloud@latest/copilot/concepts/agents/about-third-party-agents) on github.com.

Just like the [Copilot cloud agent](https://docs.github.com/enterprise-cloud@latest/copilot/concepts/agents/cloud-agent/about-cloud-agent), you can now select a model when kicking off a task:

- **Claude agent**: choose from **Anthropic** models
- **Codex agent**: choose from **OpenAI** models

This is intended to give access to the latest and most capable models as they become available.

## Available models

### Claude coding agent models

The following models are currently available for the Claude coding agent:

- Claude Sonnet 4.6
- Claude Opus 4.6
- Claude Sonnet 4.5
- Claude Opus 4.5

### Codex coding agent models

The following models are currently available for the Codex coding agent:

- GPT-5.2-Codex
- GPT-5.3-Codex
- GPT-5.4

## Enable access to third-party coding agents

- Access to **Claude and Codex** is included with your existing **Copilot** subscription.
- If you are a **Copilot Business** or **Copilot Enterprise** subscriber:
  - An administrator must enable the relevant policy for **Anthropic Claude** or **OpenAI Codex**.
  - See: [Managing policies and features for GitHub Copilot in your enterprise](https://docs.github.com/enterprise-cloud@latest/copilot/how-tos/administer-copilot/manage-for-enterprise/manage-enterprise-policies)
- The user or organization that owns the repository must enable the agent in:
  - **Settings** > **Copilot** > **Cloud agent**
