# Improved Control Over Draft Repository Security Advisories on GitHub

GitHub has introduced an update for repository administrators, allowing them to lock and unlock draft repository security advisories and private vulnerability reports. This feature is designed to give greater control over the editing process and ensure the integrity of security advisory records.

## Key Features

- **Administrator Control:** Only repository administrators can lock or unlock draft advisories; collaborators are prevented from editing the content or metadata when locked.
- **Collaborator Participation:** While the advisory is locked, collaborators can still engage via comments but cannot make direct changes to the advisory itself.
- **Workflow Enhancement:** Locking advisories can help preserve the integrity of important fields (such as severity) during triage and decision-making, ensuring no unintended edits occur during ongoing discussions.
- **Usage:** To use this feature, admins can go to the advisory and select **Lock advisory** from the advisory actions menu.

## Benefits

- **Improved Security:** Prevents accidental or unauthorized modifications to sensitive vulnerability information.
- **Enhanced Triage Process:** Allows administrators to finalize and protect advisory details while still gathering input from collaborators.
- **Auditability:** Preserves a trusted record of security advisories and private vulnerability reports throughout the workflow.
