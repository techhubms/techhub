Mario Rodriguez explains GitHub’s updated GitHub Copilot data usage policy: starting April 24, Copilot Free/Pro/Pro+ interaction data may be used to train AI models by default, with an opt-out available in Copilot privacy settings; Business and Enterprise users aren’t affected.<!--excerpt_end-->

## Summary

GitHub is updating how it uses data from GitHub Copilot to improve its AI coding assistance. Starting **April 24**, **Copilot Free, Pro, and Pro+** users’ **interaction data** will be used to **train and improve AI models by default**, unless the user **opts out**.

**Copilot Business** and **Copilot Enterprise** users are **not affected** by this change.

## What’s changing on April 24

From April 24 onward, GitHub may use Copilot interaction data to train and improve AI models, including:

- Inputs and outputs
- Code snippets
- Associated context

This is intended to make Copilot more context-aware and improve model performance.

## How to opt out (or confirm you’re opted out)

You can opt out in GitHub settings:

- Go to: https://github.com/settings/copilot
- Look under **“Privacy”**

If you previously opted out of the setting that allows GitHub to collect this data for product improvements, GitHub states your preference is **retained**, and your data will **not** be used for training unless you opt in.

## What data may be collected and used for training

If you participate (i.e., you have not opted out), GitHub lists the interaction data that may be collected and leveraged, including:

- Outputs you accepted or modified
- Inputs sent to Copilot (including code snippets shown to the model)
- Code context surrounding your cursor position
- Comments and documentation you write
- File names, repository structure, and navigation patterns
- Interactions with Copilot features (chat, inline suggestions, etc.)
- Feedback on suggestions (thumbs up/down ratings)

## What this program does not use

GitHub states the program does **not** use:

- Interaction data from **Copilot Business**, **Copilot Enterprise**, or **enterprise-owned repositories**
- Interaction data from users who **opt out** of model training in Copilot settings
- Content from issues, discussions, or private repositories **at rest**

GitHub notes that Copilot may still process code from private repositories while you’re actively using Copilot (to provide the service). That interaction data could be used for training unless you opt out.

## Data sharing scope

GitHub says the data used in this program:

- May be shared with GitHub affiliates in its corporate family, including **Microsoft**
- Will **not** be shared with third-party AI model providers or other independent service providers

## Rationale GitHub gives

GitHub says training with real-world interaction data improves model performance and that internal use of Microsoft employee interaction data has already shown improvements (including higher acceptance rates across multiple languages).
