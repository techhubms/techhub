# Workspace Monitoring dashboard templates in Microsoft Fabric Eventhouse

Fabric workspace monitoring provides telemetry across workspace assets, including **Eventhouses**, **Power BI Semantic Models**, **Data Engineering (GraphQL)**, and **Mirrored Databases**. Workspace monitoring data is stored in an **Eventhouse**, part of **Fabric Real-Time Intelligence**.

The update introduces **Real-Time Dashboard Templates** with built-in visualizations to help turn monitoring data into actionable insights. Two templates are currently available:

- **Eventhouse Monitoring Dashboard** (for Eventhouse items)
- **Semantic Model Monitoring Dashboard** (for Semantic Model items)

## Getting started

From any **Workspace Monitoring Eventhouse**, you can create these dashboards directly:

1. Open your **Workspace Monitoring Eventhouse**.
2. In the upper ribbon, select **Fabric Monitoring**.

![Workspace Monitoring Eventhouse with upper ribbon showing Fabric monitoring options to create monitoring dashboards](//dataplatformblogwebfd-d3h9cbawf0h8ecgf.b01.azurefd.net/wp-content/uploads/2026/03/the-image-shows-the-workspace-monitoring-eventhous.png)

## Available dashboard templates

### Eventhouse Monitoring Dashboard

Create the template here: [Eventhouse Monitoring](https://aka.ms/MonitorEventhouse)

Use it to monitor Eventhouse items in your workspace, including:

- Ingestion results and logs
- Commands and queries monitoring
- Metrics related to Eventhouse performance

### Semantic Model Monitoring Dashboard

Create the template here: [Semantic Model Monitoring](https://aka.ms/SemanticModelOperations)

Use it to monitor Semantic Models in your workspace using semantic model logs, including:

- Identify periods of high or unusual **Analysis Services engine** activity by:
  - capacity
  - workspace
  - report
  - user
- Analyze query performance and trends, including external **DirectQuery** operations
- Track semantic model refresh durations, overlaps, and processing steps
- Monitor custom operations sent using the **Premium XMLA endpoint**

## Customizing the dashboards

After creation, dashboards are ready to use immediately and can be edited to match your needs. For example, you can:

- Edit queries to add custom filters
- Change query logic
- Join the data with other monitoring data you have
