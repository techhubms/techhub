# Trace any Copilot coding agent commit to its session logs

Every commit from [Copilot coding agent](https://docs.github.com/copilot/concepts/agents/coding-agent/about-coding-agent), GitHub’s cloud-based background agent, is **authored by Copilot**, with the human who gave Copilot the task marked as the **co-author**. This makes it easier to identify code generated by the agent and who started the task.

## Link commits to agent session logs

The agent’s commits now link back to the agent session logs by including an `Agent-Logs-Url` trailer in the commit message.

This provides a permanent link from agent-authored commits back to the full session logs, helping you:

- Understand why Copilot made a change during code review
- Trace changes later for auditing purposes

To learn more, see the docs: [Tracking GitHub Copilot’s sessions](https://docs.github.com/copilot/how-tos/use-copilot-agents/coding-agent/track-copilot-sessions).

## Availability and admin enablement

Copilot coding agent is available to:

- Copilot Pro
- Copilot Pro+
- Copilot Business
- Copilot Enterprise

For Copilot Business and Copilot Enterprise, an administrator must enable Copilot coding agent from the “Policies” page before it can be used:

- [Enable Copilot coding agent (Enterprise Cloud docs)](https://docs.github.com/enterprise-cloud@latest/copilot/how-tos/agents/copilot-coding-agent/enabling-copilot-coding-agent)
