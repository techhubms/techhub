Allison announces two GitHub updates that surface coding-agent (including GitHub Copilot) session activity directly inside GitHub Issues and Projects, making it easier to track queued/working/review/completed status from planning views.<!--excerpt_end-->

## Overview

GitHub shipped two features that surface **coding-agent session activity** directly in **GitHub Issues** and **GitHub Projects**, so you can track agent work status during planning and execution.

These changes apply when a coding agent (examples given: **Copilot**, Claude, Codex) is assigned to work.

## Agent sessions under assignees in GitHub Issues

When a **coding agent** is assigned to an issue, a corresponding **agent session** now appears **under the assignee** in the issue sidebar.

Each session shows a live status:

- queued
- working
- waiting for review
- completed

You can click a session to open the **session logs**.

**Availability**: Generally available for repositories that have access to coding agents.

## Agent sessions in GitHub Projects table and board views

Agent sessions now also appear inside **GitHub Projects**:

- **Table view**: See which items have agent sessions and their current status
- **Board view**: See session presence/status while tracking work across columns

This is intended to make it easier to track progress across a larger set of work items.

### How to enable in a project

In your project, open the **View** menu and enable:

- **Show agent sessions**
