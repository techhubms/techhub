## Code Security risk assessment available for organizations

Organization admins and security managers can now run a free **Code Security risk assessment** to review security vulnerabilities across their organization.

## What the assessment includes

The assessment report summarizes vulnerabilities across the organization by:

- **Severity**
- **Rule type**
- **Programming language**

It also includes **remediation guidance**, and highlights where **Copilot Autofix** can automatically suggest fixes:

- [Copilot Autofix responsible use documentation](https://docs.github.com/enterprise-cloud@latest/code-security/responsible-use/responsible-use-autofix-code-scanning)

## Why it’s useful

The report is intended to help you:

- Identify **high-impact repositories** to prioritize
- Understand where to focus remediation work
- **Remediate security issues faster**, including with Autofix suggestions when available

## How to run it

To start an assessment:

1. Open your GitHub organization.
2. Go to the **Security** tab.
3. Navigate to **Assessments**.

## Availability

This feature is available in:

- **GitHub Enterprise Cloud**
- **GitHub Team**

It will also ship in:

- **GitHub Enterprise Server 3.22**
