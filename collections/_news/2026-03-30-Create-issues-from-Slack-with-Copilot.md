# Create issues from Slack with Copilot

You can now create GitHub Issues directly from Slack using natural language with the [GitHub app](https://slack.com/marketplace/A01BP7R4KNY-github) for Slack.

Mention `@GitHub` in any channel, describe the work you need to track, and the app creates structured issues in your repository. The app can also create sub-issues with proper hierarchy.

This builds on the existing [Copilot coding agent integration](https://github.blog/changelog/2025-10-28-work-with-copilot-coding-agent-in-slack/), and brings issue creation into the same chat context where teams already discuss work.

## Key capabilities

- **Natural language issue creation**: Describe work in plain language and `@GitHub` creates structured issues with:
  - titles
  - bodies
  - assignees
  - labels
  - milestones
- **Sub-issue support**: Break down work into parent and child issues from a single message.
- **Conversation mode**: Iterate on issues in a Slack thread with `@GitHub` to refine details before creation.
- **Issue flex pane**: View issues you’ve created directly from Slack by clicking on the title.
- **Channel-level instructions**: Run `@GitHub settings` to set default working repositories for the issues and pull requests the GitHub app creates in your channel.

## Try it out

1. Make sure [GitHub Copilot](https://docs.github.com/copilot) is enabled for your account. This feature is available with all Copilot plans.
2. Install or upgrade the [GitHub app](https://slack.com/marketplace/A01BP7R4KNY-github) in your Slack workspace.
3. In any Slack channel, mention `@GitHub` with a prompt like:

```text
Create an issue in my-org/my-repo to add dark mode support for the settings page
```

Then follow along as the issue is created.
