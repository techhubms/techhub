# Fix merge conflicts in three clicks with Copilot cloud agent

You can now fix merge conflicts directly on github.com using a new **Fix with Copilot** button, powered by **Copilot cloud agent**.

## How the “Fix with Copilot” flow works

1. Click **Fix with Copilot** on github.com.
2. GitHub prepopulates a comment asking Copilot to resolve the conflicts.
3. Submit the comment.

Copilot cloud agent then:

- Resolves the merge conflicts
- Checks that the **build and tests still pass**
- Pushes the updated changes

All of this runs from Copilot’s **cloud-based development environment**.

## Using `@copilot` in pull requests

You can mention `@copilot` in PRs to ask Copilot cloud agent to handle common tasks, for example:

- **Fix failing GitHub Actions workflows**:
  - `@copilot Fix the failing tests`
- **Address code review comments**:
  - `@copilot Address this comment`
- **Make other changes**:
  - `@copilot Add a unit test covering the case when the model argument is missing`

## Availability and admin enablement

Copilot cloud agent is available with **all paid Copilot plans**.

If you use **Copilot Business** or **Copilot Enterprise**, an administrator must enable Copilot cloud agent before you can use it:

- Copilot cloud agent overview: https://docs.github.com/copilot/concepts/agents/cloud-agent/about-cloud-agent
- Admin enablement (Business/Enterprise): https://docs.github.com/en/copilot/how-tos/administer-copilot/manage-for-enterprise/manage-agents/manage-copilot-cloud-agent
