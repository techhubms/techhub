# New Low Quality option in the Hide comment menu

GitHub moderators can now classify hidden comments as **Low Quality** using a new option in the **Hide comment** dropdown menu. This is available across:

- Issues
- Discussions
- Pull requests
- Commits

Previously, the available classification options (for example, **Spam** and **Abuse**) focused on traditionally problematic content. They didn’t capture the increasing volume of comments that are unhelpful or otherwise low quality.

The new **Low Quality** classifier is intended to give moderators a more accurate way to categorize these comments during moderation.

![Screenshot showing the Hide comment dropdown menu with the new Low Quality option selected](https://github.com/user-attachments/assets/d1ea20bc-92c0-41f1-8761-f782e93d8b0b)

## How to use it

1. Navigate to any comment on an issue, discussion, pull request, or commit.
2. Select the **…** menu on the comment.
3. Choose **Hide comment**.
4. Select **Low Quality** from the classification dropdown.
