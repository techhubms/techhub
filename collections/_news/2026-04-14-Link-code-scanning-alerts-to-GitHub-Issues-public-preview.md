Allison announces a public preview feature that lets teams link GitHub code scanning alerts to GitHub Issues, making it easier to track and prioritize security remediation work in existing planning workflows.<!--excerpt_end-->

## Overview

You can now link **GitHub code scanning alerts** to **GitHub Issues**, so security remediation work can be tracked through your existing planning and tracking process. This feature is in **public preview**.

## What’s new

With this update, you can:

- **Link alerts to issues**
  - From the **code scanning alert page** using the new **Tracking** section
  - From the **issue page** using the new **Security alerts** section in the **Relationships** panel
- **See tracking status at a glance**
  - Repository and organization alert lists now show **icons** to indicate which alerts are already tracked in issues
  - This helps you quickly identify **untracked** alerts that still need attention
- **Filter alerts by tracking status**
  - Use the following filters on code scanning alert lists and security campaigns:

```text
has:tracking
no:tracking
```

## Availability

This feature is available (public preview) on **github.com** for repositories with **code scanning enabled**, and on **GitHub Enterprise Cloud with data residency**.

- GitHub Enterprise Cloud with data residency: https://docs.github.com/enterprise-cloud@latest/admin/data-residency/about-github-enterprise-cloud-with-data-residency
