Microsoft Fabric Blog announces the general availability of OneLake File Explorer, a Windows File Explorer integration that lets teams drag and drop local files into OneLake so they can be used immediately in Microsoft Fabric pipelines, notebooks, semantic models, and downstream analytics workflows.<!--excerpt_end-->

## Overview

[OneLake File Explorer](https://learn.microsoft.com/fabric/onelake/onelake-file-explorer) is now generally available. It integrates OneLake directly into **Windows File Explorer**, so developers, data scientists, and data engineers can move data from local files into OneLake using familiar desktop workflows.

## From local files to cloud analytics—without breaking your flow

Typical workflow friction when source data lives on a local machine:

- Uploading files via a browser portal
- Writing scripts to push data into the lake
- Coordinating with someone else who has access

OneLake File Explorer aims to remove that friction by making OneLake appear alongside local folders in Windows File Explorer.

## How it works

With OneLake File Explorer:

- OneLake appears in **Windows File Explorer**.
- You can browse workspaces and navigate OneLake items like folders and files.
- You can **drag and drop** files from your PC directly into OneLake.

*Figure: OneLake in Windows File Explorer.*

The intent is to reduce context switching:

- No portals
- No scripts
- No tool switching

## Designed for these scenarios

OneLake File Explorer is positioned for cases where:

- You want a unified view of OneLake data across workspaces.
- Source data is local (for example: **Excel, CSV, Parquet, images, model artifacts**).
- Data needs to land directly in OneLake.
- Files should be immediately usable by:
  - Fabric pipelines
  - Notebooks
  - Semantic models
  - Downstream analytics

## Basic flow: drag and drop, then get to work

1. Open **Windows File Explorer**.
2. Navigate to your **OneLake workspace**.
3. Drag and drop files from your local machine.

After the files land in OneLake:

- Fabric pipelines can pick them up.
- Notebooks can read them without extra configuration.
- Models and semantic layers can consume them immediately.
