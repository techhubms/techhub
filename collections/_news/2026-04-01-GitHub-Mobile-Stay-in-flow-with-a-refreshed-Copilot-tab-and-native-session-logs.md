Allison announces GitHub Mobile updates that make it easier to manage GitHub Copilot agent sessions on iOS and Android, including a refreshed Copilot tab, native session logs, and controls to create and review pull requests from completed sessions.<!--excerpt_end-->

## Overview

GitHub Mobile adds new in-app capabilities to help you manage **GitHub Copilot** agent sessions while on the go, including a refreshed Copilot experience on Android, **native session logs**, and more controls for handling sessions directly from the app.

## Android: Copilot tab and refreshed Home

On Android, Copilot is now accessible directly from the navigation bar as the **Copilot** tab, making it faster to jump into:

- Your sessions
- Chat history

Within the Copilot tab, a new Home experience provides a clearer overview of agent sessions and chat history to help you find what you need more quickly.

![Android GitHub Mobile screenshot showing the Copilot tab and refreshed Copilot experience](https://github.blog/wp-content/uploads/2026/04/Android-inline-1.png?resize=1024%2C538)

## Session list view improvements

A new session list view includes:

- Filters to sort tasks by state
- Real-time updates as new sessions start

## New actions you can take in the app

You can now do more with Copilot agent sessions directly in GitHub Mobile:

- View full session logs natively
- Create a pull request from a completed session
- Review the pull request in depth
- Stop a running session

## Availability and links

These updates are available in the latest production build of GitHub Mobile:
