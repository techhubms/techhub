# GitHub Copilot Coding Agent for Jira: Public Preview

Developers can now delegate Jira issues to the GitHub Copilot coding agent, an asynchronous and autonomous AI agent that creates draft pull requests in your GitHub repository.

## How It Works

- **Context Analysis:** Copilot agent analyzes the Jira issue’s description and comments to inform its work.
- **Autonomous Implementation:** The agent implements necessary changes independently and opens a draft pull request on your behalf.
- **Jira Updates:** Get regular progress updates via Jira’s agent panel.
- **Clarifying Questions:** If Copilot needs more details, it will ask for clarifications directly in Jira.

## Key Benefits

- **Automates repetitive tasks**, including bug fixes and documentation updates.
- **Reduces context switching** between Jira and GitHub.
- **Maintains existing review and approval workflows** for all Copilot-generated PRs.

## Getting Started

1. **Install the App:** Download [GitHub Copilot for Jira](https://marketplace.atlassian.com/apps/1582455624) from the Atlassian Marketplace. Install the [GitHub app](https://github.com/marketplace/github-copilot-for-jira) during setup.
2. **Connect GitHub Organization:** Link your GitHub org during configuration.
3. **Configure Repository Access:** Specify which repositories Copilot can access in your settings.
4. **Assign Issues:** Assign a Jira issue to GitHub Copilot, or @mention the agent in comments to trigger automation.

## Requirements

- **Jira Cloud with Rovo enabled**
- **GitHub Copilot coding agent enabled**
- **A connected GitHub repository**
- Installation of both Jira and GitHub apps

Refer to the [official integration documentation](https://docs.github.com/copilot/how-tos/use-copilot-agents/coding-agent/integrate-coding-agent-with-jira) for detailed setup instructions and share feedback via the provided survey.

> This integration is available for GitHub Data Residency customers in any supported region.
