Allison shares an update that GitHub Copilot coding agent now starts work 50% faster, improving how quickly developers can hand off tasks like creating or iterating on pull requests via issues, the Agents tab, or @copilot comments.<!--excerpt_end-->

## Overview

GitHub has optimized **Copilot coding agent** so it now **starts work 50% faster**, reducing the wait time before the agent begins making changes.

## Ways to hand work to Copilot coding agent

You can ask Copilot coding agent to start work in several ways:

- Assign an issue to Copilot: [Assigning an issue to Copilot](https://docs.github.com/copilot/how-tos/use-copilot-agents/coding-agent/create-a-pr#assigning-an-issue-to-copilot)
- Enter a prompt in the repository **Agents** tab: [Asking from the Agents tab/panel](https://docs.github.com/copilot/how-tos/use-copilot-agents/coding-agent/create-a-pr#asking-copilot-to-create-a-pull-request-from-the-agents-tab-or-panel)
- Mention `@copilot` in a pull request comment: [Asking Copilot to make changes](https://docs.github.com/copilot/how-tos/use-copilot-agents/coding-agent/make-changes-to-an-existing-pr#asking-copilot-to-make-changes)

## What the coding agent does

Copilot coding agent:

- Works in its **own cloud-based development environment**
- Makes changes
- Runs your tests
- Pushes the results

## What changes with faster startup

With shorter startup times:

- Copilot can **create pull requests from scratch** more quickly
- You get a faster feedback loop when asking Copilot to iterate on an existing pull request using `@copilot`
