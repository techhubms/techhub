# Email Notifications for GitHub Usage Thresholds

GitHub now sends email alerts as your included usage nears its monthly limits across several of its metered products: Actions, Packages, Git LFS, and Codespaces.

With this update, users receive notifications at both 90% and 100% of their included allowance within a billing period. This system aims to help teams prevent surprise overage charges or unexpected workflow interruptions.

**Products covered by these notifications:**

- Actions minutes & storage
- Packages bandwidth & storage
- Git LFS bandwidth & storage
- Codespaces core hours & storage

Each notification email clearly identifies the account involved, the specific product, a usage summary versus allowance, and the billing period. It also provides a direct link to relevant budget and monitoring tools.

**Admin Controls:**

- These alerts are configurable by enterprise admins, organization admins, and billing managers.
- You can enable or disable them from the **Included usage alerts** section on the “Budgets & alerts” page in GitHub.

For more information, see the official [Included usage alerts documentation](https://docs.github.com/en/billing/concepts/budgets-and-alerts?versionId=free-pro-team@latest&productId=billing#included-usage-alerts).

These improvements provide better cost control and operational awareness for teams relying on GitHub's cloud development tooling.
