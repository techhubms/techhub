# Take your PostgreSQL-backed apps to the next level

PostgreSQL is widely used, and Microsoft is positioning it as a strong fit across its developer stack: Azure offerings for running Postgres, Visual Studio Code tooling for local development and migrations, and AI services for building “agentic” experiences.

The main challenge called out is complexity when you need to:

- Tune performance
- Design for resiliency
- Operate at scale
- Build AI/agent experiences that depend on reliable, well-modeled data access

To address this, the authors created the **PostgreSQL Like a Pro** video series:

- Series link: [https://aka.ms/postgreslikeapro](https://aka.ms/postgreslikeapro)
- Focus: practical guidance and real-world demos for PostgreSQL on Azure, using Microsoft tools from local dev through production
- Notes: highlights “supercharged by AMD technologies”

## Here’s what you’ll learn from the pros

- **Building AI agents that actually work**
  - Uses the **Model Context Protocol (MCP)** so agents can explore a database and run **vector searches** using natural language.
  - Described as working “without leaving **Microsoft Foundry**.”

- **Migrating without the manual grind**
  - Introduces **AI-assisted migration tooling in VS Code**.
  - Uses an “agentic self-correction” approach to automatically catch and fix schema or code issues.
  - Example migration path: **Oracle → Azure Database for PostgreSQL**.

- **Optimizing PostgreSQL performance on Azure**
  - Positions **Azure Database for PostgreSQL** as meeting performance and resiliency needs.
  - Mentions “fine-grained control” and “flexible deployment options.”

- **Scaling to meet the most demanding workloads**
  - Mentions a PostgreSQL offering named **Azure HorizonDB**.
  - Describes **decoupled compute and storage** as the mechanism for improved performance/scaling for mission-critical apps.

## What to expect

The series previews best practices, architecture, and outcomes when pairing **PostgreSQL on Azure** with **AMD technology**.
