Rachel Cohen shares a call for sessions for GitHub Universe (Oct 28–29) and highlights five memorable past talks—covering Git workflow tips, secure GitHub Actions patterns with Copilot, GitHub Advanced Security, Kubernetes security training themes, and Copilot Autofix-style AI-assisted remediation.<!--excerpt_end-->

## Overview

GitHub Universe returns Oct 28–29 at Fort Mason Center (San Francisco). GitHub’s **Call for Sessions** is open **now through Friday, May 1 at 11:59 p.m. PT**.

- Submit a proposal: https://reg.githubuniverse.com/flow/github/universe26/cfs/page/cfs-submissionguide
- Nominate a speaker: https://reg.githubuniverse.com/flow/github/universe26/cfs/page/cfs-submissionguide

The post also highlights five past Universe sessions as inspiration.

## Highlighted past sessions

## Git’s nine lives: Taming frontend chaos with Git’s hidden features (2025)

![Pillippa Pérez Pons on stage at GitHub Universe 2025](https://github.blog/wp-content/uploads/2026/04/Image-1.jpeg?resize=1024%2C683)

Pillippa Pérez Pons framed common frontend/Git pain points—rebases, growing monorepos, missing commits, and general Git chaos—through a “cat’s nine lives” narrative. Each “life” introduced lesser-known Git features and optimizations, including:

- sparse checkouts
- partial clones
- reflog-based recovery
- Git performance boosts

## Breath of Copilot: Level up your DevEx with GitHub Actions and GitHub Advanced Security (2025)

![Matteo Bianchi (GitHub) and Alexandra Aldershaab (Eficode) on stage at GitHub Universe 2025](https://github.blog/wp-content/uploads/2026/04/Image-2.jpeg?resize=1024%2C683)

Matteo Bianchi and Alexandra Aldershaab presented a story-driven session that maps CI/CD modernization to a fantasy quest. The practical core was:

- secure **GitHub Actions** patterns
- security-forward automation and supply chain risk awareness
- using **Copilot** as a “sidekick” in the workflow
- positioning **GitHub Advanced Security** as part of keeping security “front and center” while speeding up delivery

## Dream it in the morning, build it in the afternoon: Collapsing the distance from idea to impact (2025)

Martin Woodward discussed “speed” as a creative lever rather than a pure productivity metric, emphasizing experimentation and curiosity as software moves from idea → prototype → shipped experience faster.

## Dungeons and deployments: The clusters of chaos (2024)

A roleplaying-style session used a fantasy narrative to teach **Kubernetes security** concepts, delivered by Noah Abrahams, Ian Coldwater, Kat Cosgrove, Seth McCombs, and Natali Vlatko.

## Mission Copilot Autofix: Securing the world’s software (2024)

Nick Liffen and Niroshan Rajadurai described how GitHub applies **AI** to streamline remediation and make security alerts easier to interpret, and pointed toward AI capabilities that can strengthen defenses and potentially identify emerging threats earlier.

## Universe ’26 sessions and submissions TL;DR

For help shaping a submission, GitHub points to the submission guide:

https://reg.githubuniverse.com/flow/github/universe26/cfs/page/cfs-submissionguide

This year’s session formats:

- **Demo-style sessions**: Product demos and Ship & Tell
- **Thought leadership**: Breakout Sessions, Panels, Fireside chats (with Q&A)
- **Interactive learning**: Workshops and Sandbox sessions

Note: **Ship & Tell** is a new format aimed at builders sharing what they shipped, how it scaled, what broke, what worked, and what others can learn.
