# Copilot usage metrics now includes organization-level GitHub Copilot CLI activity

Following prior releases for enterprise-level and user-level CLI telemetry, GitHub has added **organization-level** support to Copilot usage metrics.

Organization admins can now view **CLI-specific activity** and **usage totals** in their **1-day** and **28-day** reports.

## What’s included in organization-level CLI reporting

This update adds the following CLI-focused metrics:

- **Daily active CLI users**
- **CLI session counts** and **request counts**
- **Token usage totals**, including **average tokens per request**

For details, see the documentation: [Copilot usage metrics API](https://docs.github.com/enterprise-cloud@latest/rest/copilot/copilot-usage-metrics?apiVersion=2026-03-10)

## Why this matters

- Track **CLI adoption** across your organization alongside existing **IDE metrics**.
- Understand how teams are using Copilot from the command line via **session volume** and **request counts**.
- Use **token usage breakdowns** to monitor consumption trends and plan for expanded rollout.
