# Figma MCP Server Integration with GitHub Copilot in VS Code

GitHub Copilot users now have access to a powerful new feature: direct integration with the Figma MCP server. This enhancement allows developers to pull design context directly from Figma into their codebase and send rendered UI components back to Figma as editable frames.

## Key Capabilities

- **Pull design context into code:** Use Copilot to access Figma design details and generate code based on that context.
- **Send rendered UI back to Figma:** Push your UI components from VS Code to Figma where they can be iterated as editable frames.
- **Synchronized workflow:** Move seamlessly from design to code and back, ensuring UI and code stay aligned.

## Getting Started

1. **Install the [Figma MCP server](https://developers.figma.com/docs/figma-mcp-server/remote-server-installation/#vs-code).**
2. **Connect your Figma account** as prompted.
3. **Use GitHub Copilot** within VS Code to pull design context or push rendered UIs back to Figma.

> *Note: Capturing UI to Figma as editable frames requires the use of the remote MCP server.*

## Availability

- Any developer with a GitHub Copilot subscription can use this feature.
- Available on all Figma seats and plans.
- Currently supported in VS Code; Copilot CLI support is coming soon.

For more information, visit the [Figma MCP server Guide](https://help.figma.com/hc/articles/32132100833559-Guide-to-the-Figma-MCP-server) or join the conversation in the [GitHub Community](https://github.com/orgs/community/discussions/categories/announcements).

## Summary

This feature empowers developers to maintain a tight feedback loop between design and code, reducing friction and improving developer experience when building UIs with Copilot.
