Allison announces that GitHub Codespaces is generally available for GitHub Enterprise Cloud with data residency, outlining supported regions and the ownership policies required to keep codespaces compliant with data residency requirements.<!--excerpt_end-->

## Overview

GitHub Codespaces is now generally available (GA) for **GitHub Enterprise Cloud with data residency**. This enables teams to use **secure, configurable, instantly available cloud development environments** while meeting **data residency** requirements. Codespaces for data residency is stated to have **full feature parity** with Codespaces on the general GitHub platform.

## Supported regions

Codespaces is available in all GitHub Enterprise Cloud with data residency regions:

- Australia
- EU
- Japan
- US

## Important considerations for data residency

To maintain strict data residency:

- **Enterprise-owned or organization-owned codespaces are required**.
- **User-owned codespaces aren’t supported** for GitHub Enterprise Cloud with data residency accounts.
- Configure **enterprise or organization policies** to ensure proper ownership for Codespaces usage.

More information:

- Documentation on codespace ownership: https://docs.github.com/codespaces/managing-codespaces-for-your-organization/choosing-who-owns-and-pays-for-codespaces-in-your-organization
- Learn more about GitHub Codespaces: https://docs.github.com/codespaces
- GitHub Enterprise Cloud with data residency: https://docs.github.com/enterprise-cloud@latest/admin/data-residency/about-github-enterprise-cloud-with-data-residency
