# Dataflow Gen2: Variable Library integration in Microsoft Fabric (Generally Available)

*If you haven’t already, check out Arun Ulag’s blog “FabCon and SQLCon 2026: Unifying databases and Fabric on a single, complete platform” for a complete look at all of the FabCon and SQLCon announcements across both Fabric and database offerings:* https://aka.ms/FabCon-SQLCon-2026-news

Production data integration requires more than functional transformations—it requires consistency across environments, repeatable deployments, and governance at scale. Variable Library integration is now **generally available** for **Dataflow Gen2** and enables enterprise CI/CD and ALM patterns by allowing centrally managed configuration values to be resolved at runtime.

## Production workloads (Generally Available)

This release indicates Variable Library integration for Dataflow Gen2 meets Microsoft’s standards for:

- Reliability
- Supportability
- Operational readiness

Customers can adopt it as part of production data integration solutions in Microsoft Fabric.

## Designed for real-world Dataflow Gen2 scenarios

A common challenge in data integration projects is managing configuration differences across environments (for example: dev/test/prod). Variable Library integration helps by allowing Dataflow Gen2 items to:

- Reference **centrally managed values** at runtime
- Avoid embedding **environment-specific details** directly in queries

## A first-class capability in the Fabric platform

Variable Library integration is now positioned as a first-class part of the Dataflow Gen2 experience in Microsoft Fabric. It aligns with Fabric platform principles around:

- Governance
- Consistency
- CI/CD-ready workflows

This reinforces Dataflow Gen2 as a production-ready data transformation engine.
