# Microsoft Agent Framework: The Future of AI Agent Development after Semantic Kernel

Last week, Shawn Henry announced the [**Microsoft Agent Framework**](https://aka.ms/AgentFramework), marking a key milestone for developers building agentic AI applications. Below you'll find a summary of the main topics, resource links, and practical guidance for teams and individuals invested in Semantic Kernel and AutoGen.

## Key Resources

- **Official Blog Post:** [Introducing Microsoft Agent Framework: The Open-Source Engine for Agentic AI Apps](https://devblogs.microsoft.com/foundry/introducing-microsoft-agent-framework-the-open-source-engine-for-agentic-ai-apps/)
- **Documentation:** [https://aka.ms/AgentFramework/Docs](https://aka.ms/AgentFramework/Docs)
- **Demos:** [AI Show](https://aka.ms/AgentFramework/AIShow) | [Open at Microsoft](https://aka.ms/AgentFramework/OpenAtMicrosoft)

## About Microsoft Agent Framework

Microsoft Agent Framework is positioned as the unified, enterprise-grade platform for building, deploying, and managing AI agents. It serves as the evolution of Semantic Kernel—incorporating lessons learned and feedback from the community, while also leveraging integration with Microsoft and Azure ecosystems. Developers can expect:

- Open-source development model
- Support for both C#/.NET and Python
- Deep Azure AI integration
- A clear migration path from Semantic Kernel and AutoGen

## Support for Semantic Kernel

- Microsoft Agent Framework is developed by the same team as Semantic Kernel and AutoGen and can be considered Semantic Kernel v2.0.
- **Semantic Kernel v1.x** will continue to receive support for critical bugs and security issues, as well as updates until at least one year after Agent Framework's General Availability (GA).
- Most new features will now target Microsoft Agent Framework, but Semantic Kernel will see continued maintenance as long as there is substantial usage.

## Language Support Parity

- Microsoft intends to keep Python and C#/.NET parity for features that reach General Availability.
- During preview phases, some features may launch on one platform before the other.

## Guidance for Developers

- If working on existing projects or need to ship quickly, continuing with Semantic Kernel is still supported.
- For new projects, especially those that can wait for full General Availability, Microsoft recommends starting with Microsoft Agent Framework.
- For questions or discussion, developers are encouraged to use the [Semantic Kernel GitHub Discussions](https://devblogs.microsoft.com/foundry/introducing-microsoft-agent-framework-the-open-source-engine-for-agentic-ai-apps/) and [Discord community](https://aka.ms/foundry/discord).
- Migration resources are provided for both [.NET](https://github.com/microsoft/agent-framework/tree/main/dotnet/samples/SemanticKernelMigration) and [Python](https://github.com/microsoft/agent-framework/tree/main/python/samples/semantic-kernel-migration) developers.

## Naming and Branding

Microsoft Agent Framework emerges as the official successor to Semantic Kernel, despite the community's fondness for earlier names like "Semantogen." The aim is to provide a single, consistent call-to-action for AI agent development on Microsoft technologies.
