# View code and comments side-by-side in pull request Files changed page

Docked panels for the pull request **Files changed** page are rolling out now. They let you review code with key pull request context open side-by-side:

- Overview
- Comments
- Merge status
- Alerts

This helps you stay in flow without switching tabs or losing your place.

If you’re still using the classic **Files changed** page, you’ll need to opt into the new experience to view docked panels. For details, see the changelog entry: [Improved pull request Files changed page on by default](https://github.blog/changelog/2026-01-22-improved-pull-request-files-changed-page-on-by-default/).

## Comments panel

The **Comments** panel lets you:

- View review comments and high-level conversations in one place
- Add new comments from the panel without navigating away

How to open:

- Click **Comments** in the **Files changed** toolbar

More details: [Access all pull request comments without leaving the new Files changed page](https://github.blog/changelog/2026-02-19-access-all-pull-request-comments-without-leaving-the-new-files-changed-page/).

![A pull request showing a code file diff alongside a comments panel for reviewing and discussing changes.](https://github.com/user-attachments/assets/84ab850f-fdee-4d82-937a-2705387ff95b)

## Overview panel

The **Overview** panel lets you reference the pull request description while staying in the diff. This can help when you need context like:

- Goals
- Implementation notes
- Testing instructions

How to open:

- Click **Overview** in the **Files changed** toolbar

More details: [Pull request Files changed public preview (November 20 updates)](https://github.blog/changelog/2025-11-20-pull-request-files-changed-public-preview-november-20-updates/).

![A pull request showing a code file diff alongside an overview panel for viewing the pull request description.](https://github.com/user-attachments/assets/e1ea96ca-d403-49b9-a819-3fb7f5a0483e)

## Merge status panel

**Merge status** is now available at the top of every pull request page. It’s intended to help you check merge readiness quickly by highlighting:

- Blockers
- Missing approvals
- Required checks

How to open:

- Click **Merge status** in the pull request header

More details: [Quick access to merge status in pull requests (public preview)](https://github.blog/changelog/2026-03-05-quick-access-to-merge-status-in-pull-requests-in-public-preview/).

![A pull request showing a code file diff alongside a merge status panel summarizing approvals, checks, and blockers.](https://github.com/user-attachments/assets/28621401-0766-4f7a-bd40-33a2d1055da7)

## Alerts panel

The **Alerts** panel brings security findings into the same place you review code. It surfaces **code scanning alerts** related to changes in the pull request so you can review findings alongside the diff and take action without leaving your review.

How to open:

- Click **Alerts** in the **Files changed** toolbar

![A pull request showing a code file diff alongside an alerts panel reporting a check failure.](https://github.com/user-attachments/assets/baa9e2a2-11fd-4c8c-81ee-2ea313173609)
