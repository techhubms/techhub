# Copilot Usage Metrics Now Includes User-Level GitHub Copilot CLI Activity

GitHub has expanded its Copilot usage metrics to now track user-level CLI activity. This builds on the recent introduction of enterprise-level CLI telemetry by introducing a more granular view of how individual developers are interacting with Copilot through the command line.

## Key Enhancements

- **User-Level Visibility**: Organizations can now see which developers are active Copilot CLI users. This helps spot areas where enablement or support may be needed.
- **Detailed Activity Reporting**: Track the number of CLI requests, individual session counts, and developer usage patterns over time.
- **Token Usage Monitoring**: Gain insights on token consumption trends at the user level, allowing for better resource planning and rollout strategies.

## How to Get Started

- Consult the [API documentation](https://docs.github.com/enterprise-cloud@latest/rest/copilot/copilot-usage-metrics?apiVersion=2022-11-28#get-copilot-users-usage-metrics-for-a-specific-day) for specifics about retrieving user-level metrics.
- Enterprise administrators can use this data to help drive Copilot adoption or identify where additional training is needed.
- For questions or feedback, participate in the [GitHub Community discussion](https://github.com/orgs/community/discussions/categories/announcements).
