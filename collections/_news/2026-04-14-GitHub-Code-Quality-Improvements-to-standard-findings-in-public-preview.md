# GitHub Code Quality: Improvements to standard findings in public preview

We’ve updated the GitHub Code Quality experience to make it easier to navigate and triage findings across your repository.

GitHub Code Quality **standard findings** help you detect potential **reliability** and **maintainability** issues on your repository’s **default branch**. Each finding includes a **fix suggestion generated by GitHub Copilot Autofix**, aimed at helping you resolve issues more quickly.

## What’s new

- **Search findings by file path**
  - Use the search box to filter the file list and quickly locate specific files.
- **Dismiss findings in bulk**
  - Select multiple findings and dismiss or reopen them in bulk directly from the rule page.
- **View messages and related locations for each finding**
  - See full context per finding, including the diagnostic message and related code locations.

## Availability

The GitHub Code Quality standard findings page is available to:

- Users participating in the **GitHub Code Quality public preview** on github.com
- **GitHub Enterprise Cloud with data residency** users: https://docs.github.com/enterprise-cloud@latest/admin/data-residency/about-github-enterprise-cloud-with-data-residency
