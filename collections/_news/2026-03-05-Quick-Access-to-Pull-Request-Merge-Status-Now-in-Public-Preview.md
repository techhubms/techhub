# Quick Access to Pull Request Merge Status Now in Public Preview

GitHub has announced that the pull request merge status is now prominently displayed at the top of every pull request page. This enhancement allows users to:

- Instantly check merge readiness anywhere in the pull request process, including within the new “Files changed” view
- Click to open detailed merge requirements, helping you quickly identify blockers, missing approvals, or outstanding tasks
- Streamline the workflow of preparing a pull request for merge

**Opting in to the New Experience:**

- The improved merge status feature is available on the new “Files changed” page
- Users still relying on the classic “Files changed” page must opt into the updated experience to see merge status there
- More details and opt-in instructions are available in [the changelog post](https://github.blog/changelog/2026-01-22-improved-pull-request-files-changed-page-on-by-default/)

**Feedback Opportunity:**

- Feedback is encouraged via [the feedback discussion thread](https://github.com/orgs/community/discussions/188033)
- Users are invited to report bugs, ask questions, and contribute suggestions

This feature aims to simplify collaboration, making it easier to move pull requests smoothly toward completion and enhancing transparency in code review workflows.
