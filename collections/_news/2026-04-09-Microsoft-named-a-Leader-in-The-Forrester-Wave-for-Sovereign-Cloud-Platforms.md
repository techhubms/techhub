# Microsoft named a Leader in The Forrester Wave™ for Sovereign Cloud Platforms

Microsoft was named a Leader in **The Forrester Wave™: Sovereign Cloud Platforms, Q2 2026**, which evaluated sovereign cloud providers on current offerings, strategy, and customer feedback.

- Full report: https://reprint.forrester.com/reports/the-forrester-wave-tm-sovereign-cloud-platforms-q2-2026-f64f29c7/index.html

![Figure 1 of The Forrester Wave™: Sovereign Cloud Platforms, Q2 2026](https://azure.microsoft.com/en-us/blog/wp-content/uploads/2026/04/Image_Sovereign-Cloud-Platforms-Q2-2026-772x1024.webp)

## Why this recognition matters

The post frames **digital sovereignty** as a mainstream requirement for organizations operating across borders and regulated environments. It also highlights Forrester’s point that there is **no single sovereign deployment model** that fits all needs; organizations often combine:

- Public cloud
- Private cloud
- Disconnected environments

The goal described is not “isolation,” but consistent controls across multiple environments while retaining access to modern cloud capabilities.

## A platform approach to sovereignty

The article describes **Microsoft Sovereign Cloud** as a set of capabilities applied consistently across public and private environments, combining technical controls, operational practices, and contractual commitments.

Microsoft Sovereign Cloud is described as bringing together:

- **Public cloud**
  - Data residency and access controls
  - Region-specific residency controls, including **EU Data Boundary**
- **Private cloud / hybrid deployments**
  - Enabled via **Azure Local**: https://azure.microsoft.com/en-us/products/local/
  - Consistent policy/management via **Azure Arc**: https://azure.microsoft.com/en-us/products/azure-arc/
- **Partner-operated national clouds**
  - Examples mentioned: **Bleu** and **Delos Cloud** (infrastructure independently owned and operated to meet national requirements)

The post’s claim is that this enables organizations to evolve their sovereignty posture over time without leaving the Microsoft cloud ecosystem.

- Related: Microsoft Sovereign Cloud overview: https://www.microsoft.com/en-us/sovereignty
- White paper: https://info.microsoft.com/ww-landing-microsoft-sovereign-cloud-in-europe.html

## Consistency across sovereign environments

A key differentiator cited is Microsoft’s ability to provide capabilities across sovereign public and sovereign private cloud, including:

- **Containers and Kubernetes**
- Running Kubernetes clusters in connected or disconnected environments using **Azure Arc** and **Azure Local**
- Support via **infrastructure-as-code** and **GitOps** tooling

The post argues this matters because sovereign cloud is not only about where data resides, but whether organizations can:

- Operate and secure workloads consistently across environments
- Maintain development and operations standards
- Avoid fragmenting teams, tools, and processes

## Looking ahead

The post states sovereign platforms are evolving as customers want to apply:

- AI development and runtime capabilities
- Analytics
- Modern application services

It also notes the viewpoint that customers don’t buy sovereignty as a standalone product; they “architect for it” over time.
