# Changes to GitHub Copilot plans for individuals

As shared in a recent blog post, GitHub is making changes to Copilot plans for individuals to support service reliability and a sustainable Copilot experience.

## New signups paused for Pro, Pro+, and Student

To prioritize service quality for existing paying customers:

- New signups will be paused for:
  - Student
  - Pro
  - Pro+
- Copilot Free remains open for new signups.
- Existing users can still upgrade between plans.

## Tighter usage limits for individual plans

- Pro+ offers more than **5×** the limits of Pro.
- If you need higher limits on Pro, you can upgrade to Pro+.
- As users approach their limit, they will see warning notifications in:
  - VS Code
  - Copilot CLI
- Usage progress tracking is “coming soon.”

For details on how usage limits work, see: https://docs.github.com/en/copilot/concepts/rate-limits

## Opus models removed from Pro

- Opus models are no longer available on **Copilot Pro**.
- **Opus 4.7** remains available on **Pro+**.
- As previously announced, **Opus 4.5** and **4.6** will also be removed from **Pro+**.

Related announcement: https://github.blog/changelog/2026-04-16-claude-opus-4-7-is-generally-available/

## If these changes don’t work for you

GitHub notes these changes are intended to provide a predictable experience for existing customers.

If you hit unexpected limits or the changes don’t work for you:

- You can cancel your Pro or Pro+ subscription.
- You will not be charged for April usage.
- Contact GitHub Support between **April 20 and May 20** to request a refund:
  - https://support.github.com/
