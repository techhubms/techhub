Allison announces a GitHub update that lets developers batch-apply code scanning alert fix suggestions directly in a pull request’s Files changed tab, reducing repeated scans and speeding up remediation and review.<!--excerpt_end-->

## Summary

GitHub code scanning alerts on pull requests can now be addressed faster using bulk actions.

## What changed

You can now apply fixes for code scanning alerts in a pull request’s **Files changed** tab by adding multiple alert suggestions to a batch.

## Why it matters

Batching changes into a single commit means:

- You run **one** code scanning run instead of a separate scan for each individual alert fix.
- Remediation and review time can be reduced.
- Pull requests can move through review faster.

## Availability

This update is now generally available on **github.com**.
