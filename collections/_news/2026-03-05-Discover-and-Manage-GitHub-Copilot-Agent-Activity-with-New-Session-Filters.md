# Discover and Manage GitHub Copilot Agent Activity with New Session Filters

GitHub Enterprise AI Controls and the agent control plane have rolled out several enhancements to help enterprises monitor and manage agent activity more effectively.

## What’s New

- **Status Filter**: You can now filter agentic sessions based on their status—such as queued, in progress, completed, failed, idle waiting for user, timed out, and cancelled.
- **Repository Filter**: Filter agent sessions by the specific repository where the activity took place, allowing for more targeted oversight.
- **User Filter**: Search for sessions by the user who initiated the agent session, simplifying user activity tracking across your enterprise.

These new capabilities supplement existing filters, such as searching by agent and organization, and aim to give enterprises detailed insights into agent usage patterns, operational status, and management needs.

For more information, see [AI controls and the agent control plane documentation](https://docs.github.com/copilot/how-tos/administer-copilot/manage-for-enterprise/manage-agents/monitor-agentic-activity).
