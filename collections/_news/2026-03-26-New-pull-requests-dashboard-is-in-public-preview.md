Allison announces a public preview of GitHub’s refreshed pull requests dashboard at github.com/pulls, adding an inbox, saved views, and improved filtering to help developers organize and prioritize review work.<!--excerpt_end-->

## Overview

A public preview of the refreshed pull requests dashboard is now available at [github.com/pulls](https://github.com/pulls). The update introduces:

- A new pull request **Inbox**
- **Saved views** to organize common queries
- More **powerful filtering** and advanced search capabilities

## 

## 

## 📥 Inbox

**Inbox** is designed to highlight pull requests that need your attention, including:

- Pull requests needing your **review**
- Pull requests that **need fixing**
- Pull requests that are **ready to merge**

You can filter by:

- **Repository**
- How recently a pull request was **updated**

![Pull requests inbox with review requests, needs action, and ready to merge sections](https://github.com/user-attachments/assets/949bd324-3178-4f2e-b93c-15a911322f56)

## 📂 Saved views

Saved views let you create, edit, and organize custom views based on your most-used search queries.

![Pull requests dashboard focused on a saved view, in an edit state](https://github.com/user-attachments/assets/650cb657-4a1c-4289-9329-2bc07ec0e503)

### Create a saved view

To create a saved view from an existing view:

1. Click the **“+”** (**Create a view**) icon in the sidebar
2. Give the view a name
3. Click **Save view**

## 🧐 Powerful filtering

New filtering options aim to make it easier to find the pull requests that matter.

- **Smart default filters**: Built-in filters include:
  - `Authored by me`
  - `Assigned to me`
  - `Involves me`
  - `Review requested`

  Notes:
  - `Authored by me` includes *all* of your pull requests, including those authored by **GitHub Copilot** on your behalf.

- **Filtering assistance**: Content assist and auto-complete help find pull requests across:
  - Repositories
  - Teams
  - Organizations
  - Projects

- **Advanced search support**:
  - Use `AND` and `OR`
  - Use nested searches
  - A space between filter fields acts as an `AND`

Example:

```text
(org:github AND author:@me) OR (org:dizzbot assignee:mona)
```

This finds pull requests across multiple organizations.
