# SQL Database Enhancements in Microsoft Fabric: Copilot, Data Virtualization, Notebook Integration, and Backup/Restore

**Presented by Anna Hoffman | Microsoft Developer**

This Data Exposed episode provides a comprehensive look at recent and upcoming enhancements to SQL databases in Microsoft Fabric with the goal of improving development, analytics, and operational workflows.

## Topics Covered

### 1. Copilot Integration for SQL in Fabric

- Overview of Copilot capabilities in SQL databases
- How Copilot assists with query authoring and workflow automation
- Demo resources: [MS Learn documentation](https://aka.ms/fabric-sql-copilot), [Demos](https://aka.ms/fabric-sql-copilot-demos)
- [Introducing Copilot for SQL Database in Microsoft Fabric](https://blog.fabric.microsoft.com/en-us/blog/introducing-copilot-for-sql-database-in-microsoft-fabric?ft=Databases:category)

### 2. Data Virtualization (Preview)

- What data virtualization is and how it helps access disparate data sources
- Preview features for virtualizing external data in SQL databases in Fabric
- Resource: [MS Learn documentation](https://aka.ms/fabricsqldv)

### 3. Notebook Integrations

- Using notebooks directly with SQL databases in Fabric
- Benefits for data engineering and collaborative analytics
- T-SQL magic commands and how to leverage them
- Resource: [MS Learn documentation](https://learn.microsoft.com/en-us/fabric/data-engineering/tsql-magic-command-notebook)

### 4. Backup and Restore Improvements

- New capabilities for simplifying backup and restore workflows
- How these enhancements improve reliability and disaster recovery
- Resources: [Backup](https://learn.microsoft.com/en-us/fabric/database/sql/backup) | [Restore](https://learn.microsoft.com/en-us/fabric/database/sql/restore)

## Key Takeaways

- Enhanced developer productivity with Copilot features and notebook integration
- Easier management and access to data using data virtualization
- Improved operational resilience through advanced backup and restore
- All features designed to streamline SQL workflows in Microsoft Fabric
